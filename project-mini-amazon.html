Overview

Mini-Amazon is a full-scale e-commerce web application built from the ground up, modeling the core functionalities of Amazon. Sellers can create product listings and manage inventories; buyers can browse, search, purchase products, and leave reviews. All transactions are conducted using virtual currency within the platform. The project was developed by a team of 7 students over the course of a semester, with each member responsible for a distinct functional domain while collaborating closely on database design and frontend integration.

My Role: Sellers Guru (Inventory & Order Fulfillment) & Team Leader. I designed and implemented the complete seller inventory management system and order fulfillment pipeline. Additionally, I coordinated cross-team collaboration, managed project milestones, and led the integration of all five functional modules into a cohesive platform.

My Contributions: Sellers Module

Seller Dashboard

  • Built the seller dashboard at /sellers/<uid> that displays a seller's complete inventory by querying the Inventory and Products PostgreSQL tables via parameterized SQL.
  • Implemented conditional UI rendering: the "Seller Dashboard" button appears in the navigation bar only when a user has the vendor role, handled via Flask session management.
  • Dashboard displays product name, price, current stock quantity, and links to edit each product.

Product Add & Edit

  • Implemented the product addition endpoint at /seller/add that handles adding new products and auto-registering the user in the Sellers(uid) table if they haven't already been registered.
  • Built the inventory editing endpoint at /seller/edit/<uid>/<pid> that updates quantities for existing items in the Inventory table.
  • Designed the product creation form (name, price, quantity) and editing form with server-side validation and error handling.

Order Fulfillment

  • Implemented /seller/orders that lists every open line item for the seller with buyer information, order date, total amount, and overall fulfillment status.
  • Built fulfillment and cancellation actions that update order statuses without double-modifying inventory (since order submission already decrements inventory atomically).
  • Added buyer statistics surfacing on the orders page to help sellers prioritize outreach to repeat customers.

Database Schema (Sellers Domain)

  • Sellers(uid) — identifies users registered as sellers, with cascading FK to Users.
  • Products(pid, name, price, available) — stores product listings with unique name constraints.
  • Inventory(sid, pid, quantity, price) — composite key (sid, pid) links sellers with products; maintains per-seller pricing and stock integrity.
  • Used CHECK constraints on price and quantity columns to prevent negative values.

卖家仪表盘

商品添加与编辑

  • 实现了/seller/add商品添加端点，处理添加新商品并在用户尚未注册时自动将其注册到Sellers(uid)表。
  • 构建了/seller/edit/<uid>/<pid>库存编辑端点，更新Inventory表中现有商品的数量。
  • 设计了商品创建表单（名称、价格、数量）和编辑表单，包含服务端验证和错误处理。

订单履约

  • 实现了/seller/orders，为卖家列出所有待处理订单项，包含买家信息、订单日期、总金额和履约状态。

数据库设计（卖家域）

  • Inventory(sid, pid, quantity, price) — 复合主键(sid, pid)关联卖家与商品；维护每个卖家的定价和库存完整性。
  • 在price和quantity列上使用CHECK约束防止负值。
